1 - Named Structure

1.1 - Base usage

A structure is a group of variable that represents data. It is a user define custom type, defined with the keyword struct in packages, namespaces, classes, or function . It is composed of attribute; variables for structured type. As structure can't implement methods; function for structured type, it is not needed to add the keyword var, but the declaration of the attributes follow the same rule than variable declared with var.

package main

struct type Position = {
	x, y int
}

func main() {
	var p Position
	p = Position{x: 18, y: 42}
	p2 = Position{}
}

"Position" is a new type with the attribute "x" and "y" of type int. There is automatically 3 way to declare a value of a named structure. The first is by setting all attribute by name (Position{x: 18, y: 42}), or simply in order (Position{31, 99}), but it's not possible to mix them. The second option is with the var variable declaration, which will basically set the same value than the third way; by calling the default constructor (Position{}).

It is possible to assign default value to the attributes, in such case the default constructor will set those value but will let the variable equal to default. So it is for the attributes ! 

package main

struct type Position = {
	x int
	y = 42 // deduced as int
}

func main() {
	var p Position
	trueExp := p == default // true
	trueExp = p.x == default && p.x == 0 // true
	trueExp = p.y == default && p.y == 42 // true
}

To access an element of a structured type, we pass through the accessor '.'

1.2 - init, get, static

1.2.1 - init and delete

It is possible to make constructors. A constructor is use to initialize the object (an object is a instance of a structured type). As saw in 1.1, there are 2 constructor by default. The empty one, and the the one whom set the value by name. It is possible to change the behavior of those constructor, and even to add constructor. A constructor is represented with the keyword init followed by the value it must take.

package main

import Std io.console as cio

struct type Contry = {
	name string
	lang string
	init() = default
	init(name, lang string) = default
	init(name string) = default {
		.lang = "english" // to refere attribute while inside the structured type, 
											// mark "self.ATTRIBUTE" or simply ".ATTRIBUTE"
	}
	init(language int) {
		.lang = language as string
	}
}

func main() {
	England := Contry{"England"}
	cio.Println(England.lang) // print "english" on the console
}

Here we define new constructors whom take only one element. We add the existing constructor, but it is not mandatory. We can see the = default after constructors, it is possible to let nu-cmd fill the structure with constructors that takes parameter of the same name and type than the structured type attribute. It is still possible to complete it with code inside a scope ({}); that the case for init(name string). The init(language int) can't be default because its parameter(s) don't match any attribute. The prototype of a constructor must be unique, it means it is not possible to have 2 constructor that take the same parameters type list. With structure it is also possible to use the typed rvalue ! But unlike simple type, it must be define. A rvalue constructor must take only one parameter and is represented with a '!' after the init.

package main

import Std io.console as cio

struct type Contry = {
	name string
	lang string
	init() = default
	init(name, lang string) = default
	init!(name string) = default {
		.lang = "english"
	}
}

func main() {
	contry := "England" Contry // possible due to the 'init!'
	cio.Println(contry.lang) // print "english" on the consle
}

it is still possible to use that constructor without RValue in the classic way. The opposite of the init is the delete. It can appear only ones in a structured type, and it is call when the object is deleting.

package main

import Std io.console as cio

var expression_number = 0

struct type Expression = {
	typename string = "void"
	init() = default {
		expression_number++
	}
	init(typename string) = default {
		expression_number++
	}

	delete {
		expression_number--
	}
}

func main() {
	var exp1 = Expression{"int"}
	cio.Println(expression_number) // print 1 on the console
	{
		exp2 := Expression{}
		cio.Println(expression_number) // print 2 on the console
	}
	cio.Println(expression_number) // print 1 on the console
}

1.2.2 - the get attributes

Attributes are accessible from out of the structure, however, it is possible to define attributes that will be in a read only mode. Its the get attributes. Get attributes can't be modify out of the structure, it will take all its sense with the chapter of Extensions and Interfaces. Here how we define get attributes:

package main

var instances = 0

struct type Sample = {
	get id int
	name string
	init() {
		.id = instances++
	}
	init(name string) = default {
		.id = instance++
	}
	delete {
		instance--
	}
}

func main() {}

Here, everybody can access the attribute id, but it's only when we are in the structure that we can also modify it.

1.2.3 - static member

Static element of a type is an element whom is bind to the type, and not to an instance.

package main

import Std io.console as cio

struct type Human = {
	static get number = 0
	name string
	age int
	static init {
		.number++
	}
	delete {
		.number--
	}
}

func main() {
	var h Humain
	{
		h2 := Humain{name: "toto", age: 42}
		cio.Println(Humain.number) // 2
	}
	cio.Println(Humain.number) // 1
}

"number" is a static attribute, to call it we mark the type (Humain) dot the attribute. As it is marked as get it is readable but not writable from out of the structure. We use "number" in the delete, because all static element are usable inside the structured type. But we also see static init. static init is something call before all constructor. There is only one and it can access only static elements, so impossible to have .name in the static init here.

2 - Lambda Structure

Lambda structure are lambda without name that can be define everywhere. However, it is not possible to define constructor, destructor, static element not get attributes.

2.1 - Lambda structure type

package main

func main() {
	var someData struct{id int, name string}
}

A lambda structure is identified by its attributes name and type, but not regarding its location, that means, two variable can have the same lambda structure type even if they are not at the same location.

package main

func A() {
	var LambdaA struct{_ int}
}

func main() {
	var LambdaB struct{_ int}
}

the structure in the function A is the same type than the structure of the function main. Therefore, it is possible to assign one to another. By the way, we see that it is possible to define unnamed attribute(s). To access those attributes, we must use the accessor operator followed by the number of the attribute.

package main

func main() {
	var sample struct{_, _ int}
	sample.1 = sample.2 // the second attribut is assigned to the first one
}

2.2 - Lambda structure value

Named structure are lambda structure with... a name, so every named structure has an implicit cast to the lambda structure it is made of. That behavior can't be changed.

package main

struct type A = {
	age  int
	name string
}

func main() {
	var data struct{age int, name string}, data2 A = {18, "Glycine"}
	data = data2 // ok !
}

However, it is not possible to cast them as pointer (we'll see pointer later). Because it would break the get spec'. static attributes aren't keep in the cast. A lambda structure can be represented directly by a RValue.

package main

func main() {
	data := struct{age: 42, name: "Magnolia"}
}

data is a lambda struct of type: struct{age int, name string}.

2.3 - Lambda structure quickly

As lambda struct may be really useful, there is shortcut to define them:

package main

func main() {
	var data1 {{age int, name string}}
	data2 := {{age: 42, name: "Cyclamen"}}
}
