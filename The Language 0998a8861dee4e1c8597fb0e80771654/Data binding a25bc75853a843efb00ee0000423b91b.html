1 - attribute binding aka Structure binding

It is possible to bind structures attributes to variables with the data binding

package main

import Std io.console as cio

func main() {
	data := {{name: "Newton", discovered: "Gravity"}}
	{Scientist: name} := data
	cio.Println(Scientist) // will print 'Newton"
}

Here the attribute 'name' of the variable 'data' (which is a lambda struct: struct{name, discovered string}) is "bind" to the variable 'Scientist'. It is possible to bind as many variable as attribute, and the order doesn't matter. It is possible to bind element without specifying the name, if the name of the variable match the attribute one.

package main

struct type Human = {
	name  string
	age   int
	alive bool = true
}

func main() {
	Benj := Human{name: "Benj", age: 25}
	var age int
	{age} = Benj // only get the age of Benj.
}

Structure's attributes can also be accessed through their position, so it is also possible to bind their position, that way also works with static size arrays !

package main

struct type Human = {
	name  string
	age   int
	alive bool = true
}

func main() {
	Benj := Human{name: "Benj", age: 25}
	var BenjAge int
	{BenjAge: 2} = Benj // only get the age of Benj.
	// BenjAge = Benj.2
	data := ["Hello there !", "General Kenobi"] // data is [2]string
	{answer: 1} := data // answer = data[1]
}

It is even possible to bind structure attributes to another structure attributes, and even creating a lambda structure with the attribute we didn't take.

package main

struct type Human = {
	name string
	age int
	alive = true
}

func main() {
	Maxime := Human{name: "Maxime", age: 30}
	{
		var name string, other {{ age int, alive bool }}
		{name, other...} = Maxime
	}
	// or
	{
		{name, other...} := Maxime
	}
	
}

Here, other will be filled by the attributes of Maxime that are not already taken by the variable before. In the first case, if other didn't have all the other attribute name, it would have filled only the one he got. Note that doing: {other...} := Maxime is equivalent to write: other := Maxime. The attribute binding also allows to create a new lambda structure, it is called the destructuring !

package main

func main() {
	s1 := {{a: 18, b: "Camille"}}
	s2 := {{...s1, c: true}} // <=> s2 := {{a: s1.a, b: s1.b, c: true}}
}

The only difference between attribute binding and attribute destructuring is the ... comes before (for the destructuring).

2 - order binding aka Array binding

That way of binding is more large, it occurs on container and structures. It binds according to the order of the data, first will go to the first, second to the second, etc.

package main

func main() {
	array := [2, 4, 6, ...]
	[first] := array // <=> first := array.first
	data := {{ObiWan: "Hello there", Grievous: "General Kenobi !"}}
	[obiSays, grivAnsw] := data
	// obiSays = "Hello there"; grivAnsw = "General Kenobi !"
}

If we want to skip elements before the wanted value, we simply put the _.

package main

func main() {
	colors := ["blue", "white", "red", "grey"]
	[_, secondaryColor] := colors
}

even if its calls the Array-binding that binding also works with structures

package main

func main() {
	structure := {{key: "first", value: 1}}
	[k, v] := structure // k = structure.1, v = structure.2
	// <=> {k: 1, v: 2} := structure
}

Same logic than structure binding, it is possible to create an array and destructure one.

package main

func main() {
	arr := [1, 3, 5]
	[first, others...] := arr // <=> first = arr[0], others = [arr[1], arr[2]]
	
	arr2 := [...arr, 7, 9] // <=> arr2 := [arr[0], arr[1], arr[2], 7, 9]
}

3 - Argument binding for function call

We are going to see function just after. But now, just know that we call a function or a method with parentheses, and we pass arguments in between. It is possible to bind arguments

package main

func MyFunc(arg1, arg2 int) { // 2 arguments: arg1 and arg2 typed int
	// code
}
func MyFunc2(arg1 int, arg2 string) { // same but arg1 is int, arg2 is string
	// code
}

func main() {
	{
		args := [18, 31, 23]
		MyFunc(args...) // <=> MyFunc(args[0], args[1])
	}
	{
		args := [18, 31, 23]
		MyFunc([args...]) // <=> MyFunc(args...)
	}
	{
		args := {{arg1: 18, arg3: 31, arg2: 23}}
		MyFunc({args...}) // <=> MyFunc(args.arg1, args.arg2)
		args2 := {{arg1: 18, arg2: "Lovely number"}}
		MyFunc2({args2...})
	}
	
	// unsafe, those may throw at runtime
	{
		args := ["arg1": 18, "arg2": 31]
		MyFunc({args...}) // <=> MyFunc(args["arg1"], args["arg2"])
		// throw if args key don't match the functions arguments name

		MyFunc([args...]) // <=> MyFunc(args.values...)
		// throw if args.length is less than number of the function argument 
		
	}
	{
		args := [2, 3, 6, ...]
		MyFunc([args...]) // <=> MyFunc(args[0], args[1])
		// throw if args.length is less than number of function argument
	}
}

It is not possible to control which data will be passed, for array binding it is the firsts element, and for structure binding it is the named element. It is possible to create the lambda/array directly in the function call, and bind it to the arguments. In that case, there is no need to surround the value by {}/[] to bind them.

package main

func MyFunc(a, b int) {}

func main() {
	MyFunc({{a: 18, b: 23}}...)
	/*
	<=>
	{
		tmp := {{a: 18, b: 23}}
		MyFunc({tmp...})
	}
	*/
	MyFunc([18, 23]...)
	/*
	<=>
	{
		tmp := [18, 23]
		MyFunc([tmp...])
	}
	*/
}
