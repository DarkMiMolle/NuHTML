1 - Extend your types, add method

Extension are a way to "extend" a given type by adding methods ( and athibutes?) to it.

package main

import Std io.console as cio

extension type int = {
    const func square() => self * self 
}

func main() {
    cio.Println( 5.square ) 
}

You noticed here that we called square without parentheses. It's because it's a getter like method. A getter method must take 0 parameter and return a non void value. It is still possible to use parentheses to call it. Actually it may be necessary, if we return a function we want to call in place for example: 

package main

import Std io.console as cio

extension type int = {
	func IncrementFn() func() void => func => self++
}

func main() {
	a := 18
	inc := a.IncrementFn // ok, inc is func()void
	// <=> inc := a.IncrementFn()
	inc() // a now has value 19
	a.IncrementFn()() // ok, call the returned function in place, a = 20.
}

Remarque: it won't be possible to mark 18.IncrementFn because it is not a const func (know more in the next chapter)

2 - Import extension

To import an extension from a package or a namespace, it only needs to import that package or namespace. Then to apply the extension, we need to specify it.

package main

import {
	Std TypeWrapper.Number // extends the type int 
	Std io.console as cio
}

extension type int: TypeWrapper.Number.int = {
	// all function must be implemented
	const Square() int -> super[0] // use the implementation of its super[0] type
	// super[0] == first of the super type aka TypeWrapper.Number.int
	// ...
	// implementation of all methods of TypeWrapper.Number.int
}

func main() {
	cio.Println(5.Square)
}

It is possible to want the extension of multiple package/namespace... Its the same way to do it. So if there is any conflict it will be resolve naturally. In all case, super[0] represent the first type after the ':', super[1] the second, super[2] the third, etc... instead of writing super[i] it is possible to write the location of the extension directly.

// Example/Pkg1.nu.pkg/ext1.nu

package Pkg1

extension type int = {
	const Square() => self * self
	const Double() => self * 2
	SelfSquare() Self => self *= self // Self indicate it return itself
}

// Example/Pkg2.nu.pkg/ext2.nu

package Pkg2

extension type int = {
	const Square() => self * self
	const Double() {
		if self == 0 then return 2
		else return self * 2
	}
	const Triple() => self * 3	
}

// Example/main.nu.pkg/main.nu

package main

import {
	Pkg1
	Pkg2
}

extension type int: Pkg1.int, Pkg2.int = {
	const Square() int -> super[0] // <=> Pkg1.int
	const Double() int -> Pkg1.int // auto resolve the conflict with Pkg2.int
	const Triple() int -> super[1] // <=> Pkg2.int
	SelfSquare() Self -> Pkg2.int // <=> super[1].int
}

3 - extend type when creating one

During the chapter of the type we saw the with keyword that allows us to declare the default value of a new creating type. It is also possible to combine with extension. When so, the extension it defaulted to the type.


package Pkg1
import Std io.console as cio

type MyInt = int with extension {
	const Print() => cio.Printl(self)
}
public alias mi -> MyInt

// other file
package main
import Pkg1

func main() {
	val := 42 Pkg1.mi
	val.Print() // print 42
	// no need to re implement the extension
}

We can omit the in place definition of the extension to indicate that type will be with the extension by default (if in the same package)

package Pkg1
import Std io.console as cio

type MyInt = int with extension
public alias mi -> MyInt

extension type MyInt = {
	const Print() => cio.Print(self)
}

This Pkg1 is strictly equivalent to the previous juste before.
