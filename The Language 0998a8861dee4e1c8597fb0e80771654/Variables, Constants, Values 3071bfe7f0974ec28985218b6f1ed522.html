1 - Variables

Variables are a mandatory aspect of the nowadays programming language. It is the way to store the result of an expression, and use it.

1.1 - Declaration

Every variable has a type, and can only store value of the same type.

package main

func main() {
	var variable int
}

Here, "variable" is... a variable of type int which is one of the built in type of Nu representing an integer. In Nu every variable are initialized, to avoid storing random unknown value, but it is also possible to initialize variable manually.

package main
func main() {
	var realNumber float = 18.31
}

We've declare the variable "realNumber" of type float to the value "18.31" (float is a builtin type representing a floating point number) Now, it would be convenient to be able to declare multiple variable, of multiple type. like:

package main
func main() {
	var v1, v2 string, v3 int = 42, v4 char = 'N'
}

"v1" and "v2" are both string(builtin type representing a text: string of character), "v4" is a char (builtin type representing a character). So the type applies to all variables untyped yet. That's why "v1" is string. The other rule to do that kind of declaration is that before the first user-initialization, all variable must be typed, and after the first user-initialization, every variable must be user-initialized. If a variable is user-initialized, it is possible to not specify the type and let the language infer it automatically.

package main
func main() {
	var v1, v2 int, v3 = "this is a string"
}

"v3" will be infer as a string. However, the language will infer the most simple type possible, so var a = 42 will never give a float. When user-initialized, it is also possible to remove the var keyword, and use the declaration operator ':='.

package main
func main() {
	v1 := 42 // v1 declared with value 42 (int)
}

It is still possible to declare multiple variable with that way:

package main
func main() {
	v1, v2, v3 := "string, int, char", 18, 'N'
}

One last thing. So far all example show the use of variable inside a function (func main() { /*...*/}), but it is possible to declare variable in the Global Space meaning directly in the package. However, those variable must be declare with the keyword var.

package main
var global = 42 // impossible to use ':='
func main() {
	noGlobal := global
}

1.2 - Assignment

The principe of variables is they are... well variable. So it is possible to assign them value even after declaration.

package main
func main() {
	var i int
	i = 31
}

At first "i" has the int default value (which is 0), and then we assign "31" to "i" so "i" has the value "31". It is possible to assign multiple value to multiple variable at the same time.

package main
func main() {
	var v1, v2 int
	v1, v2 = 42, 24
	v1, v2 = v2, v1
}

At first we declare "v1" and "v2" as int, then we assign them respectively 42 and 24, so "v1" = 42 and "v2" = 24. Then v1, v2 = v2, v1. Yes, of course, a variable is considered like a value of the type mentioned by the variable. So here, "v1" = "v2" (which is 42) and "v2" = "v1" (which is 24). That line is a swap of those two variable.

1.2 - Default value ?

As said earlier, every variable non user-initialized have a default value. Sometimes, it may be useful to know when the variable is the default value or not. That's why it is possible to check it.

package main
func main() {
	var v1 int, v2 = 0
	if v1 == default {
		// ...
	}
	if v2 != default {
		//...
	}
}

here, v1 == default and v2 != default are boolean expressions (meaning expressions of type bool, a builtin type for the values true and false), both equal to true, because "v1" is the non user-initialized, so it is default, and reversely for "v2". The declaration with the 'declare' operator will never give a variable default.

What is interesting to notice, is that v1 == v2 would be true because the default value of int is 0 !

2 - Constants

2.1 - Global

Constants are like variables, except they must be user-initialized and they can't be assigned. The define operator not allows to define constants neither.

package main
const c int = 42
func main() {
	v := 18
	const c2 = v
}

It is possible to use type inference, and even assign it from a variable. After that, a constant represent the value it holds (but can't change its value unlike variable). It is also possible to declare multiple constant of multiple type.

package main
const a = 18, b = 31
func main() {
	const c, d = 42, b
}

A constant can't change value.

2.2 - Scoped constants

Rules for constants inside a scope have a particularity. They can be assign ones.

package main
const cst = 42 // Global constant, must be user-initialized
func main() {
	const cst2 // Declare, but not usable !
	cst2 = 18 // now cst2 is usable
}

"cst2" is inside the function scope, so it is possible to only declare it; however, it won't be usable until Nu knows it has been initialised.

We'll see later with if-branchement, than a constant must be initialised in all branchement to be use after, or in no-one.

3.1 - Typed RValue

RValues aka Right Values, are literal values: values written in hand, not store in variable nor constant yet. 42 is an int RValue, "rval" is a string RValue. It is possible however to user-type those value. By calling the constructor of the type, or by adding the type name after the value.

package main
func main() {
	var val = int{23} // 1
	val2 := 18 float // 2
}

With "1" we use the constructor of the type. It is represented with the type followed by value(s) surrounded by '{}'. If it's a little blurry, don't worry, we'll re-speak of constructor with the chapter Structures. With "2" we are typeing the RValue directly, so "val2" is a float. This work, only because it exists a link between int and float; we'll see that link in the Chapter Type and Casts.

3.2 - RValue storage (aka const expression)

Now comes the final kind of storage. It is a way to "store" the RValues. Actually, it won't really store them, it will directly emplace the RValue it contains. RValue must be known at the compile time, so the expression must be static (not depend of the execution)

package main
func main() {
	const! val = float{42}
	variable := val - 28 // <=> variable := float{42} - 28
}

RValues are part of the "constant expression", which is expression where the value can be known at the compile time. Here the compiler knows that "val" has the value "float{42}". That kind of const: const! are called "constexpr". A constexpr can store only constant expression, meaning expression with only others const!.
