1 - Operators in struct, interface and types

1.1. Define the operators

Operators are limited. Some are user defined allowed like '+', '/'. Some must followed rule to be implemented like '[]'. And some others are not overridable like '=', '&()'. To define an operator we use the operators block. And in that block the definition of operators followed rules. First, the Self type must appear in the operand. Then the syntax looks like: - Unary operator: ( Operand Self Operator ) ReturnType or (Operator Operand Self) ReturnType - Binary operator: (Operand1 Type1 Operator Operand1 Type2 ) ResultType Then the definition of the operation must come. The Unary operator that can be overloaded are: - (Self++) Self or void the increment operator. They mustn't be implemented, but the operator (Self + int) is required as it will be automatically implemented as: self + 1. - (Self--) Self or void same except it is decrement operator and make: self - 1. - (OP Self)R : with OP one of: - and R = Self, ! and R = bool The binary operators that can be overloaded are: - (T1 OP T2) T3 where at least one of T1 or T2 is Self (T3 can also be Self) with OP equal to one of: + , - , / , * , %. Then, (T1 OP= T2) void can be defined (as default only) if (T1 OP T2) T3 is defined with T3 = T1. The operation made are then: self = self OP value (self is T1, value is T2). The operators &, = are a bit more specific. & and = do not follow the same rule of definition. They are only surrounded by parentheses: (&) and (=). Then they are followed by -> and if we are in an interface then it must be a type, else it must be an attribute. if the operator (=) is overload then the operator (&) must be overloaded as: (&) -> self (or (&) -> Self for interface)

package main
import Std io.console as cio

struct implem A = {
	val int
}
struct type A = {
	operators {
		(&) -> .val
	}
} with extension {
	const Result() => .val * .val
	const Test() {
		cio.Println(&self, .val)
	}
}

struct implem B = {
	val int
}
struct type B = {
	operators {
		(=) -> .val
		(&) -> self
	}
} with extension {
	const Print() => cio.Println(.val)
}


func main() {
	a := A{val: 42}
	cio.Println(a.Result) // 42^2 
	a -> 18
	cio.Println(a.Result) // 18^2
	a.Test() // 18 18
	// &a === a.val
	b := B{val: 19}
	b = a.Result
	(&b).Print() // 18^2
	b -> B{val: 20}
	(&b).Print() // 20
}

1.2. In structures, interfaces and extension

1.2.1. structures

Let's see directly an example:

package main

import {
	Std io.console as cio
	Std TypeAddon
	Std TypeAddon.math as math
}

cast fraction to string is explicit -> TypeAddon

struct type Complex = {
	get re fraction = 0
	get im fraction = 0
	
	init(val int) {
		.re = val
	}
	init(re, im fraction) = default
	init(re fraction) = default
	init(val float) {
		.re = val as fraction
	}

	operators {
		(l Self + r Self) Self = default
		(l Self - r Self) Self = default
		(l Self * r Self) Self {
			.return.re = (l.re * r.re) - (l.im * r.im)
			.return.im = l.re * r.im + l.im * r.re
		}
		(l Self / r Self) Self {
			.return.re = l.re / r.re + l.im / r.im
			.return.im = l.re * r.im + r.re * l.im
		}
	}
} with cast && extension = {
	const Module() => math.Sqrt(math.Pow(.re, 2) + math.Pow(.im, 2))
}

cast int to Complex is implicit // use the init of Complex
cast float to Complex is explicit // use the init of Complex
cast fraction to Complex is explicit // use the init of Complex
cast Complex to string {
	cast.to = "(" + cast.from.re as string + " + " + cast.from.im as string + "i)"
} is explicit

func main() {
	c1, c2 := Complex{5, 10}, Complex{2 \ 5, 1\3}
	cio.Println(c1 + c2 as string)
	// output: (5.4 + 31\3i) ou (27\5 + 31\3i) ça va dépendre de l'implem de TypeAddon
}

It is possible to use default implementation for operator if all attributs inside the structure have the same operator. Here, fraction has the operators: +, -; so the default behaviours for the operator + of the structure Complex will be: .return.re = l.re + r.re and .return.im = l.im + r.im. Basically it will apply the operator to each corresponding attributes by doing: Left + Right.

1.2.2. interfaces

With interfaces the operators mustn't be defined. It is a way to force type that implement it to define operator.

package main

interface type Number = {
	operators {
		(Self + Self) Self
		(Self - Self) Self
		(Self / Self) Self
		(Self * Self) Self
	}
}

func main() {
	n := 42 as Number // ok, int as all operator implemented !
	n2 := (try n as int)? + 18 ?? 0 // ok, n2 will be == to 42 + 18 = 60
	// impossible to write: n + 18 directly !
}

Wow, that special... In an interface type, the Self identifier refers to the dynamic type of the interface. So when Self identifier is used in an interface type, the method can't be call directly from the interface, but it ensures the type whom implement the interface has that method. To refer the actual interface type, we have to put it in letter so here is would be: Number. But if we had the operator: (Self + Number) Self, then 42 as Number wouldn't work because int doesn't implement the Number interface.

1.2.3. extensions

To stay on the interface explication, let's see how we do with extension.

package main

import Std io.console as cio

interface type SumNumber = {
	operators {
		(Self + Number) Self
		(Self - Number) Self
	}
}

extension type int: SubNumber = {
	operators {
		(l Self + r Number) Self {
			return try l + r as int ?? l
		}
		(l Self - r Number) Self {
			return try l - r as int ?? l
		}
	}
}
extension type float: SubNumber = {
	operators {
		(l Self + r Number) Self {
			return try l + r as float ?? l
		}
		(l Self - r Number) Self {
			return try l - r as float ?? l
		}
	}
}

func main() {
	var n SumNumber = 42 // ok, int implement SubNumber
	n2 := n + 18 // ok, 18 is int whose implement SubNumber !
	n2 = n + 18.42 // ok! float is SubNumber, float as int is valid !
	cio.Println(n2) //ouput: SumNumber{int = 60} 
}

1.3. In created type

To add operator to a created base type, we simply pass through the extension ! The existing operators can't be redefined.

2 - Make your array type

To make a type usable like an array, but whose don't come from an array (i.e: ≠ type MyArray = [5]int) there are some rule to respect. Then the operator [int] will be available. It is simple, the type must implement the methods: length that would represent the size of the array, begin, end, at(i int) each one returning an array-iterator where: &array.begin + i == at((&array.begin).index + i), &array.end - i == at((&array.end).index - i) with i an integer from 0 to length - 1. The iterator should also overload the operator the operator = to be consider as the type of the array elements. Here is the interface of the array-iterator: 


interface type ArrayIterator<type T> = {
	const Next() ArrayIterator<T>
	const Prev() ArrayIterator<T>
	const Index() int // or similar to
	const Value() *T
	
	operators {
		(=) -> *T
		(&) -> Self // mandatory to override operator =.

		(Self + int /* or similar to */) Self
		(Self - int /* or similar to */) Self
		(Self++)
		(Self--)
	}	
}

Let see an example:

package A
import Std Algorithm as algo

struct type SArray = {
	init(val, size int)
	init(size int)
} with extension

struct implem SArray = {
	arr [...]int
	init(val, size int) {
		.arr := for i := 0 to! size do -> val // online for-exp
	}
	init(size int) {
		.arr := for i := 0 to! size do -> 0
	}
}

extension type SArray = {
	append(val int) bool {
		if algo.Find(val, .arr).ok then return false
		.arr.append(val)
		algo.Sort(&.arr)
		return true
	}
	
	reduce(val int) {{ok bool, elem int}} {
		var p algo.Index
		if {ok, pos} := algo.Find(val, .arr); ok {
			p = pos
			.return.ok = true
			.return.elem = .arr[pos]
		} else then return false, -1
		.arr[p] = .arr.last
// exp of .arr [0, 2, 3, 18, 6, 10, 15, 4]
		.arr.reduce()
		algo.Sorts.bubulle(&.arr + p) // bubulle sort is the most efficient in that case
	}

	begin() => .arr.begin
	end() => .arr.end
	at(i int) => .arr.at(i)
	length() => .arr.length
}

Here, we define a new type as structure and we can use the operator [int] on its values. Remarque: actually we could have juste write: type SArray = [...]int with extension { ... } and limite the extension to the definition of append and reduce.

3 - Make your dictionary type

It is the same with dictionary [] operator. Here the interface of its iterator

interface type DictionaryIterator<type K, V> = {
	const Prev() DictionaryIterator<T>
	const Next() DictionaryIterator<T>
	const Index() int /* or similar to */
	const Key() *const K
	const Value() *V
	
	operators {
		(=) -> *V
		(&) -> Self // mandatory to override operator =.

		(Self + int /* or similar to */) Self
		(Self - int /* or similar to */) Self
		(Self++)
		(Self--)
	}
}
