1 - What is a Function

1.1 - Basic definition

A function is a code that will be call manually when developing. It takes parameter and may return value(s), both together define the prototype of a function. In Nu functions are also a type define by its prototype. To start executing code, a programme need a start function, which is the function named main by default. This function can have different prototype according to the Sub nu used. A function can be define in package, namespace and other (we'll see that later). In a package or namespace, functions must have a name, that will be use to call them.

package main

func Function(parameter float) int {
	return (parameter * 10) as int
}

func main() {
	variable := Function(4.2)
}

"Function" is the name of the function, "parameter" is a parameter, when inside the function, we call it an argument. The parameters definition is the same as one line variables, except we don't put the keyword var. It is possible to not have parameter like in the "main" function. Then we put the return type (here for "Function" it's an int). Finally we put the code in a scope which is call the body of the function. The type of that function is: func(float) int

1.2 - One line function

It is possible to make function in one line (if there is only one statement or expression). In that case, the syntax is a little different. First it is then not mandatory to mark the return type, then instead of making a scope we put => and the one line code without using the return keyword. Code that can't be declaration.

package main

func Function(parameter float) => (parameter * 10) as int

func main() {
	variable := Function(4.2)
}

The one line function have another particularity; they are inlined in the code.

2 - Lambda Function

It is possible to manipulate functions as value, so it is possible to have variables and constants of functions. 

package main

import Std io.console as cio

func square(val int) => val * val

func main() {
	transformation := square // var transformation func(int)int = square
	cio.Println(transformation(5)) // print 25
	transformation = func(val int) => val + val
	cio.Println(transformation(5)) // print 10
}

It is possible to use the syntax with scope: func(val int) { return val + val } The type of function is func(parameter types)return type; the name of the parameters don't matter. Lambda have a particularity. They can manage the capture of element from the current scope. By default a lambda capture the whole scope by constant reference, meaning we can use variable of the scope and change it, and if they are changed out of the function their value will also change inside the function. But it is possible to specify the kind of capture we want.

package main

func main() {
	var a int, b string, c bool
	f := func(&, val int) { // <=> func(val int) ...
		a = val
	}
	f(42) // now a = 42 !
	f2 := func(&const) => b == "Yes"
	if f2() then f(18) else then f(31)
	// f2() == false (b == "") then a = 31
	b = "Yes"
	if f2() then f(18) else then f(31)
	// now b == "Yes" so f2() = true then a = 18

	f3 := func(=, val int) => if c then -> f(val) 
	// f3 will never do something cause c == false at the creation
	c = true
	f3 = func(=, val int) => (if c then f(val))
	// now f3 will always call f(val) !
	// by calling f(val) it will change the value of a !!
	f4 := func(!) => 42 // nothing is captured
}

with f3 we saw something important. Even if a lambda is captured by copy (=), if itself capture by reference, it will keep the references. it is possible to specify behaviour for only some variables.

package main

func main() {
	var a, not int, b, c, d string
	f := func([&, = a, &const b, c, ! not], val int) => (val * a) as string + b + c + d
	/* in those condition: 
		a by copy
		b and c by constant references
		not is not captured
		d as specified by the first: by reference (aka the default, we could have omitted it)
	*/
}

the [] is needed to indicate it will be more than one capture specificity. If we only wanted to capture a by copy and other by default (aka reference) we could have written: func(= a, val int) => ... But as soon as there is 2 specificity, we need to use the brackets. The element itself is capture by copy after it has been created (so impossible to call it while creating it) One last things about lambda function. If we create a lambda without parameters, and with default capture, we can omit the parentheses.

3 - Multiple return type, volatile structure

functions can return multiple values !

3.1 - Multiple return as structures

By returning a lambda structure, it is possible to return multiple value at once, Nu handle it well and so simplify the return.

package main

func MultipleReturn() {{a, b int}} {
	return 18, 31 // <=> return {{a: 18, b: 31}}
}

func main() {
	{a} := MultipleReturn() // <=> a := MutlipleReturn().a
}

Top | Bottom

3.2 - the return value

So fare we've returned value with the return keyword. It is possible to use the return as a value that will be returned. To use return as value we simply need to add a '.' before the return when using it, then it is no more mandatory to return value.

package main

func MultipleReturn() {{a, b int}} {
	.return.a = 23
}

func main() {
	{a} := MultipleReturn() // <=> a := MutlipleReturn().a
}

This kind of return can't be used with the one line functions.

3.3 - volatile structure

Volatile structure are structure that can't be stored. They stay volatile, so impossible to use the return value as a value.

package main

func VolatileReturn() {{int, int}} {
	// impossible to access .return
	return 14 18
}

func main() {
	// impossible to write: r := VolatilReturn()
	[start, end] := VolatileReturn() // ok, volatile structure is not stored
	firstWW := VolatileReturn() as {{start, end int}} // ok volatile struct not stored
}

Cast from a volatile structure to a structure is explicit if the structure get all values, and else deleted. It is possible to bind volatile structure to other function argument:

package main

func NextWW(start, end int) {{start, end int}} {
	.return = {{start: start + 29, end: end + 25}}
	if .return.start >= .return.end then return 0, 0 // we stil can use the return
}

func WW(start, end int) {{int, int}} => return start, end

func main() {
	// impossible: NextWW(WW(14, 18))
	SecondWW := NextWW(WW(14, 18)...) // ok, we use the binding
}

to be clear, it is not possible to use the "structure binding" aka "name binding" because there isn't name.
