1 - Static size array

1.1 - Single size array

Static size array are array with a well defined size. That size can't change.The size must be known at the compile time, but it is not mandatory.

package main

func main() {
	var Array [4]string // array of 4 string
	var Array2 []string = ["one", "two", "three", "four"] // array of 4 string too !
	// by the way: ["one", "two", "three", "four"] is an RValue for static array
	Array3 := [0, 1, 2, 3, 4, :+5] // Array3 is [10]int !
}

When the value isn't set, or if the value doesn't fully represent the size of the array, the size must be given. It is possible to assign an array of a shorter length, but it will reset the elements behind the short size to their default value

package main

func main() {
	var Array [4]string = ["Hello", "There"] // ok, but Array[2] and [3] == ""
}

It is also possible to assign array of a larger length, but this time it will only pick the first elements of that array.

package main

func main() {
	var Array [2]string = ["Hello", "There", "General", "Kenobi"] // ok == ["Hello", "There"]
}

1.2 - Multi-Dimention array

Usually, when we want to make a multi-dimensional array, we simply make an array of array. With static length array, is possible to do that, but Nu also allows to make "real multi-dimensional" array

package main

import Std io.console as cio

func main() {
	var Matrix [5, 5]int
	cio.Println(Matrix[2, 2])
}

Matrix is an array of 2 dimension, equivalent to [5][5]int, except here, we can access element like a real 2-Dimensional array and not by taping Matrix[2][2]. This is only possible because of the static size. It is still possible to not specify the size at the declaration

package main

import Std io.console as cio

func main() {
	var Matrix [[][]]int = [[1, 2, 3][2, 4, 6][3, 6, 9]]
	cio.Println(Matrix[2, 2])
}

The differences between an RValue of [3][3]int and [3, 3]int is there is no ',' between value for the [3, 3]int.

package main

import Std io.console as cio

func main() {
	realMulti := [[1, 2, 3][2, 4, 6][3, 6, 9]] // [3, 3]int, no ',' between arrays
	ArrArr := [[1, 2, 3], [2, 4, 6], [3, 6, 9]] // [3][3]int, ',' between arrays
	cio.Println(Matrix[2, 2])
}

2 - Dynamic size array

With dynamic's arrays, it is possible to append and remove element to the end of the array, during the execution, so the size isn't known at the compile time.

package main

func main() {
	var coursesList [...]struct{product string, quantity int}
	coursesList.append(
		{{product: "chocolat", quantity: 3}}, 
		{{product: "Gameboy", quantity: 2}}
	)
	courseList.reduce() // reduce the list by deleting (and returning) the last value
}

The Rvalue of a dynamic size array is

package main

func main() {
	dynArr := [1, 2, 3, ...]
}

it is possible to reserve memory:

package main

func main() {
	dynArr := [1, 2, 3, ... :+5]　// dynArr.length == 3 ! but its capacity is 8.
}

3 - Operations on array and Slice type

3.1 - Slice of an array

A slice is a sub-array, it represents a section of an existing array. Its type is dynamic, but can be applied on static and dynamic size array.

package main

import Std io.console as cio

func main() {
	staticArr := [1, 2, 3, 4, 5]
	dynArr := [1, 2, 3, 4, 5, ...]
	var slice [&]int // the type representing a slice
	slice = staticArr[0 to 2] // slice.length == 3 (from 0 to 3 (include))
	cio.Println(slice[0]) // print 1
	slice[0] = 42
	cio.Println(staticArr[0]) // print 42 !!
	slice = dynArr[2 to! 5] // slice.length == 3 (from 0 to 5 (exclude))
	cio.Println(slice[0]) // print 3 !
}

the to is inclusive, meaning [0 to 2] is the interval [0,2][0, 2][0,2]﻿, where to! is exclusive, meaning [0 to! 5] is the interval [0,5[[0, 5[[0,5[﻿. For dynamic size array, it is possible to take slice bigger than the array, it will simply append default value to it. The default value for a Slice is nullptr.

package main

import Std io.console as cio

func main() {
	var arr [...]int
	slice := arr[0 to 18]
	cio.Println(arr.length) // print 19
}

It is possible to take slice from a slice. Even a bigger slice if the original array allows it.

package main

import Std io.console as cio

func main() {
	var arr1 [...]int, arr2 [18]int
	slice := arr1[0 to 18]
	slice2 := slice[0 to 20] // append 2 element to arr1 !
	slice = arr2[1 to 5]
	slice2 = slice[0 to 10] // ok, take a larger slice from arr2
	// slice2 = slice[0 to 18] will fail at runtime
	// because the slice wanted is bigger than the arr2 which has a static size.
}

3.2 - Operation of arrays

There are more operation on array. .length to get the length of the array [static,dynamic,slice][static, dynamic, slice][static,dynamic,slice]﻿ Warning ⛔️ with multi-dimension static array, we must specify which length we want

package main

func main() {
	staticArr := [1, 2, 3]
	multiDim := [[1, 2, 3][4, 5, 6]]
	dynArr := [1, 2, 3, ... + 1]
	trueVal := staticArr.length == 3 // staticArr.length is a constexpr !

	trueVal = multiDim.length[0] == 6 // [0] == full size (constexpr)
	trueVal = multiDim.length[1] = 2 // [1] == 1st dimension (constexpr)
	trueVal = mutliDim.length[2] = 3 // [2] == 2nd dimenstion (constexpr)
	// etc for all dimensions

	trueVal = dynArr.length == 3 // not constexpr
	dynArr.append(4)
	trueVal = dynArr.length == 4 // that's why it's not constexpr
	// actually .length is constexpr for [...]T if the whole value is constexpr
	const! arr = [1, 2, 3, ...]
	// arr.append is not possible
	// arr.length is constexpr...
}

The + operator will return a new array with element from left and right (so with a new size). It is possible to mixte dynamic and static size array, but in that case the result will be a dynamic size array. dynamic size array also have the += operator that will increase their size. [static,dynamic][static, dynamic][static,dynamic]﻿

.fill to fill the content of the array of a certain value [static,dynamic,slice][static, dynamic, slice][static,dynamic,slice]﻿

package main

func main() {
	var just42 [42]int
	just42.fill(42)
	var dyn42 [...]string
	dyn42.fill(42, "42")
	var notJust42 [42]float
	noJust42[5 to! 42].fill(42) // slice from 5 to 41 (include)
}

if there is only one element given to fill, it must be of the same type than array's elements. It is possible to give an integer before the value, that represent the number of value to set (starting at 0). For static size array it will fail if the number is bigger than its length; for dynamic array it will append the number wanted. It is possible to fill only from a certain index, to do so, simply use the slices.

.reverse is a particular slice of the array which is reversed [static,dynamic,slice][static, dynamic, slice][static,dynamic,slice]﻿

package main

func main() {
	arr := [1, 2, 3, 4, 5]
	arr.reverse[0] = 25 // arr[4] == 25
}

its type is Std core.array.ReversedSlice

.append(T)/.reduce() are method to append to the list or to reduce the list. It append/reduce to the end of the array. [dynamic][dynamic][dynamic]﻿

.first/.last *T are pointer (we'll see that later) to the first/last element of the array [static,dynamic,slice][static, dynamic, slice][static,dynamic,slice]﻿

package main

import Std io.console as cio

func main() {
	arr := ["Anakin", "Darth Vador"]
	cio.Println(arr.first, arr.last, separator: ", ") // print Anakin, Darth Vador
}

.begin/.end/at(i int) are iterator accessor. They return an iterator of type: Std core.array.Iterator. [static,dynamic,slice][static, dynamic, slice][static,dynamic,slice]﻿

package main

import Std io.console as cio

func main() {
	arr := [2, 4, 6, 8, 9, 13, 14]
	it := arr.begin // it.value == 2, it.index == 0
	it = arr.at(4)  // it.value == 9, it.index == 4
	it -> 10        // it.value now == 10, it.index == 4
	it = it.next    // it.value == 13, it.index == 5
	it -> 14        // it.value now == 14, it.index == 5
}

An iterator (for array) has the methods/getter:

.value *T a pointer to the value of the array .index int the index of the value .next/prev Std::core.array.Iterator<T> the iterator corresponding to the next/prev value (meaning the next/prev index in the array) The operator (self -> T) void that change the value hold by the iterator The operator ( &self ) *T that get the address of the value hold by the iterator we'll see more of iterator later.

all those operation are read-only (they are in get).
